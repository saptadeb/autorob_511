I. Introduction

The objective of this assignment is to study and implement various search algorithms and develop collision-free path in a JavaScript/HTML5 environment. Path planning in general is a computational problem to find a valid set of configurations that moves an object from source to goal location in a pre-defined map. In this assignment we have implemented two types of uninformed search algorithms, namely Breadth-first search and Depth-first search, and two types of informed search, namely A-star search and Greedy-best-first search. The results achieved from the path planning implementation are then compared at the end of this report.

II. Methodology

The four types of search algorithms implemented are all examples of priority first search, the difference being the computation of the priority of each node. The pre-defined map utilised in the assignment is assumed to be split into grids of 'eps' (0.1) distance. The nodes are expanded only in 4 directions (out of 8 possible directions), up, down, left and right. The neighbors of the node are then added to a 'visited' list which is sorted based on the priority of the search algorithm.

Breadth-first search (BFS) is one of the two uninformed search algorithms implemented in this assignment. BFS utilises a queue data structure which has a FIFO (first-in-first-out) rule for priority setting of a node. All the nodes at the 'd' level are expanded before the 'd+1' level. The next algorithm implemented is the Depth-first search (DFS), which is implemented as a general tree-search where the queue has a LIFO (last-in-first-out) rule for priority setting. In this algorithm the deepest node of the frontier are expanded first.

The next two search algorithms implemented are known as informed search algorithms, as the priority of the nodes are related to a heuristic value. Heuristic of a node can be defined as the estimated path cost from a node 'n' to the goal node. In case of Greedy-best-first search, the priority of a node is given as the heuristic of the node, i.e. f(n) = h(n), whereas in case of A-star search the priority of a node is given as the summation of the heuristic of the node and the path cost of the node form the start node, i.e. f(n) = g(n) + h(n). For uniformity the start and the goal positions are kept constant throughout the different planning scenes and the search algorithms, as [0,0] and [4,4].

2. Depth First Search

Figures 7-11 demonstrate the depth first search implementation, in case of the five different planning scenes provided for the assignment. Depth-first search is not expected to be complete or optimal. It has a time-complexity of O(bm) and space complexity of O(bm) , where b is the number of branches of a node and m is the minimum number of levels required to find the path to the goal. As compared to the exponential search complexity of breadth first search, the space complexity of depth first search is linear in nature because of which we expect the search iterations to be considerably smaller than that in the breadth first search.

Breadth-first search and A-star is expected to be optimal which can be corroborated by the path length in each of the planning scenes. But, breadth-first search tends to be more memory expensive than the A-star algorithm, as A-star utilises a heuristic based search which limits the number of nodes being expanded in the search. In case of depth-first search, it can be observed that the number of iterations are significantly lower than the other uninformed search (breadth-first search), but the caveat is a formulation of a sub-optimal/non-optimal path. Whereas in case of greedy-best-first search the path length is observed to be the shortest required length with the lowest number of iterations than the other three search algorithms. Even though the path formulated is the shortest it is not a optimal or complete search algorithm, as it can get trapped in a cycle (for example an obstacle, or a local minima).
